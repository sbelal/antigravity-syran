# Syran Repo Onboard and Audit Workflow

This workflow executes the `repo-onboard` skill to map out the repository and runs a logical analysis to audit and criticize any suboptimal design decisions or bad architectural choices in the workspace.

## Step 1: Scan and Map Codebase
Execute the `repo-onboard` skill to scan the repository structure and compile a detailed onboarding document outlining the components.
// turbo

## Step 2: Critical Logic Audit
Critically analyze the onboarded layout and configuration files. Identify bad practices, architectural issues, lack of separation of concerns, or redundant packages.
Write a detailed logical critique highlighting all suboptimal decisions in the repository.
